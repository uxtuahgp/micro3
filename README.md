## Домашнее задание по теме Микросервисы - подходы ##  

### Задача 1: Обеспечить разрабтку ###  
Многие продукты на сегодня предлагают как облачное, так и On-Premises развертывание.  
Можно рассмотреть как MS Azure DevOps, так и GitLab, коммерческие версии которых предоставляют примерно сходный набор инструментов.  
Наиболее широко распространным решением, отвечающим требованиям является GitLab.  
- GitLab имеет бесплатную версию с ограничениями в возможностях настройки CI/CD и платную полнофункциональную версию.  
- Из самого названия следует, что в основе лежит Git.  
- В продукте имеется возможность хранения множества проектов.  
- Система предоставляетразвитый инструментарий для построения CI/CD с помощью собственных агентов сборки runner-ов.  
- Система позволяет параллелить как сборку, так и тестирование.  
- Запуск сборк возможен как по событиям, так и по кнопке с передачей параметров.  
- Храненеи секретов можно организовать как снаружи, например в Vault или Openbao, так и внутри с помощью GitLab Secrets Manager.  

### Задача 2: Логи ###  
Стандартом для сбора, обработки и анализа логов стал стек ELK. 
- С помощью Filebeat и Logstash можно собирать логи как из stdout, так и из лог файлов.
- Logstash гарантирует доставку логов до БД Elasticsearch.
- Kibana предоставляет интерфейс для анализа логов, с настройкой аутентикации и авторизации пользователей.

### Задача 3: Мониторинг ###  
Под поставленную задачу лучше всего подходит Prometheus + Grafana, хотя с опреденными натяжками задачи можно решить и спомощью Zabbix.
- Prometheus с помощью различных exporter- ов позволяет собирать как метрики производительности ОС, так и кастом метрики СУБД, Web серверов, менеджеров очередей и прочих распространенных сервисов.
- Если требуется собирать метрики, например контейнеров, можно воспользоваться экспортером Google cAdvisor. 
- Так как prometheus exporter-ы публикуют метрики в понятном формате через HTTP, для разработчиков на любой платформе не представляет проблемы публиковать кастом метрики собственных приложений.
- Собственный интерфейс Prometheus позволяет выполнять запросы по собранным данным.
- Графический интерфейс для обработки данных, собранных Prometheus предоставляет Grafana с помощью представлений которой можно в удобном формате агрегировать статусы и ключевые метрики всех своих сервисов на одной странице.
